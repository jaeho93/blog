github.io에 ghost 기반 블로그 배포하기-1-

by Jaeho Cho

2018-07-05

들어가며

GitHub.io(GitHub Pages)는 github에서 제공해주는 무료 웹 호스팅입니다. GitHub 저장소의 내용을 웹 페이지로 서비스해주는 방식으로 동작합니다. 이를 이용하면, 어렵지 않게 자신의 블로그를 만들 수 있습니다. 블로그를 호스팅 하는데 사용하는 플랫폼으로 여러가지가 있습니다. 예를 들어, GitHub Pages에서 내부 엔진으로 사용하는 Jekyll이 있고 제가 다루고자 하는 Ghost가 있습니다. 제가 보았을 때, Ghost의 장점이라고 한다면,

  1. 툴의 UI가 직관적이다.
  2. MarkDown Editor가 편리하다.
  3. 커스텀이 용이하다.

정도를 이야기 할 수 있습니다. Jekyll과 Ghost의 차이점들을 자세히 알고싶다면 아래의 참고 문서를 참고하시기 바랍니다. 이 주제를 다루는 순서는 다음과 같습니다.

  1. 블로그를 제작하기 위한 환경 셋팅하기
  2. buster를 이용하여 GitHub 저장소에 배포하기
  3. Ghost 간단하게 사용해보기

저는 Windwos 10 운영체제를 사용중이고, OS로 ubunru 16.04 LTS를 사용하시는 분은 WSL 부분은 건너 뛰시면 됩니다. 앞으로 진행될 내용은 직접적인 프로그래밍을 다루고 있지는 않지만 익숙하지 않을 경우 어려울 수 있습니다. 또한 저의 정보 정리글로 작성하는 것이기 때문에 Ghost의 설치와 직접적으로 연관되지 않은 부분이 다소 포함되어 있습니다.

설치하기

먼저 VS Code가 설치되어 있지 않다면 여기에서 설치합니다.

다음으로 윈도우즈에서 리눅스 설치 - WSL를 참고하시어 윈도우에서 가상으로 ubuntu를 구동하는 WSL를 설치합니다 버전은 꼭 16.04 LTS로 설치합니다. 그리고 ubuntu의 bash shell 에서 다음 과정을 수행합니다.

# 빠른 다운로드 속도를 위해 mirror 사이트를 변경합니다.
~$ sudo vi /etc/apt/sources.list
# 명령어 모드에서 다음을 실행합니다.
:%s/archive.ubuntu.com/mirror.kakao.com/
:%s/security.ubuntu.com/mirror.kakao.com/
:wq

# 우분투 패키지 관리자를 업데이트 합니다.
~$ sudo apt-get update
~$ sudo apt-get upgrade

# python, pip, buster를 설치합니다.
~$ sudo apt-get install python2.7 python-pip
~$ pip install buster

# Node.js 6.X 버전과 npm을 설치합니다.
~$ curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -D
~$ sudo apt-get install -y nodejs
~$ sudo apt-get install npm

# windows에서 C:\ubuntu 디렉터리를 생성하고 ubuntu bash shell에서 다음 명령을 수행합니다.
~$ ln -s "/mnt/c/ubuntu" /home/{리눅스 사용자명}/ubuntu
~$ cd ~/ubuntu

# Ghost의 패키지를 다운로드하고 설치합니다.
~$ curl -L https://github.com/TryGhost/Ghost/releases/download/0.11.12/Ghost-0.11.12.zip -o ghost.zip
~$ sudo apt install unzip
~$ sudo mkdir /var/www/
~$ sudo unzip -uo ghost.zip -d /var/www/ghost
~$ cd /var/www/ghost
/var/www/ghost$ sudo npm install --production
/var/www/ghost$ sudo npm start --production

배포하기&실행하기

위의 과정을 수행하면 로컬에서 서버가 돌아갑니다. 이제 웹 브라우저에서

http://localhost:2368/ghost/

로 접속하여 관리자 겸 작성자 계정을 생성합니다. 여기까지 되었으면 이제 우리의 블로그를 호스팅할 레포지토리를 github에 만들어 줍니다. 저의 경우 다음과 같습니다.

https://github.com/jaeho93/blog

이제 서버가 실행중인 shell은 그대로 두고, 새로 ubuntu 프로그램을 실행하고 에 다음과 같은 과정을 수행합니다. 첫 줄의 명령어을 작성하면 repo URL을 입력하라고 나옵니다. 위의 repo 주소를 입력하고 [enter] 치면 성공했다고 나옵니다.

~$ cd /var/www/ghost/
/var/www/ghost$ sudo buster setup
Enter the Github repository URL:  
https://github.com/jaeho93/blog  
All set! You can generate and deploy now.

이제 준비가 모두 완료되었습니다. 다음의 명령어를 수행할 때마다 로컬에 있는 나의 블로그의 소스코드를 GitHub Repository에 배포하여 블로그의 정보를 업데이트합니다.

/var/www/ghost$ buster generate
/var/www/ghost$ buster deploy

deploy 할 때 는 GitHub의 계정과 비밀번호를 요구합니다, 꼭 입력해주어야 실행됩니다. 성공적으로 실행 후에 github 레포지토리에 가보면 파일들이 업로드되었습니다. 이제

https://jaeho93.github.io/blog/

와 같이 본인계정이름/github.io/레포지토리이름 으로 접속하면 블로그가 나옵니다. 만약 저와 같은 trophy-ghost theme을 사용하고 싶다면 trophy-ghost 에 접속하여 자료를 .zip 파일로 받은 후 ghost의 운영자 콘솔 화면에서 settings>general>upload theme 에 .zip 파일을 끌어다 놓으면 자동으로 적용됩니다.
